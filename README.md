# Robust OFI LOBSTER

Reproducible research pipeline for robust short-horizon return prediction from multi-level LOBSTER limit order book samples.

The project builds 500ms event-time features from five NASDAQ tickers, constructs cross-asset graphs, and compares ERM, feature-robust PGDA, and graph-robust PGDA.

## Key result

Graph-robust training gives the cleanest stability improvement in the current one-day LOBSTER sample: it reduces graph-perturbation MSE degradation and lowers graph sensitivity across all tested horizons. Feature-robust training is more mixed and mainly helps at longer horizons, with a stronger clean-accuracy/robustness trade-off.

## Data contract

| Item | Value |
|---|---:|
| Tickers | AMZN, AAPL, GOOG, INTC, MSFT |
| LOB levels | 10 |
| Time grid | 500ms |
| Bins | 46,800 |
| Feature tensor | `(46800, 5, 51)` |
| Split | chronological 70 / 15 / 15 |

Raw LOBSTER files are not committed. Place the five sample zip files in `data/raw/` before running the pipeline.

## Quick start

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# put LOBSTER sample zip files under data/raw/
bash scripts/run_pipeline_to_step06.sh
```

Each module writes a benchmark report to `reports/benchmarks/` and a gate file to `.pipeline_state/`.

## Pipeline

| Step | Purpose |
|---|---|
| 01 | raw LOBSTER audit and cleaning benchmark |
| 02 | 500ms feature tensor construction |
| 03 | graph construction and ERM baselines |
| 04b | calibrated feature-adversarial evaluation |
| 05 | graph-robust PGDA benchmark |
| 06 | final ablation summary and figures |
| 07 | GitHub packaging and safety checks |

Full scripts are in `scripts/pipeline/`.

## Project layout

```text
data/                 local data only; raw files ignored
scripts/pipeline/     benchmark-gated shell modules
src/lobster_ofi/      Python implementation generated by the pipeline
reports/benchmarks/   PASS/FAIL reports for each step
reports/final/        report-ready tables and summaries
reports/figures/      generated plots
docs/                 reproducibility and project-structure notes
```
